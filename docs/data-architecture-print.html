Thai Farmer LINE LLM Advisory Agent — Data Architecture

Farmers chat in Thai over LINE with a guardrailed LLM agent. Domain questions are answered only from a curated FAQ knowledge base (RAG) with citations; admins review every interaction.

Anti-Misinformation Controls (data-level)

  • Grounded-only answers: domain replies must carry ≥1 citation; else blocked.
  • Confidence gate: retrieval score below threshold → clarify/refuse, never hallucinate.
  • High-risk blocklist: pesticide dosage, medical, legal, finance → safe template + escalate.
  • Published-only retrieval: only status=published within valid_from/to.
  • Source attribution: authoritative source (e.g. DOA/RD) surfaced in answer.

Governance — PDPA (Thailand)

  • Consent: purpose-based, versioned; captured at LINE onboarding.
  • Minimization: store pseudonymous line_user_id; no phone/national ID.
  • Erasure: cascade delete messages/media/embeddings; audit retained lawfully.
  • Access/RBAC: roles viewer/editor/dpo; PII view is logged.
  • Residency: prefer in-region LLM+stores; redact PII before external model calls.
Key risks → hallucinated agronomic advice (→ grounded-only + citations); stale seasonal FAQ (→ validity dates + review); PII leak to external LLM (→ redaction + DPA); webhook replay (→ signature + idempotency).
